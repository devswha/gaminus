## Что делает Gajae App

Gajae App — это панель управления для одного пользователя, предназначенная для агентов программирования, работающих на вашем компьютере или сервере. Она объединяет обнаружение проектов и сеансов, потоковый чат, обработку подтверждений, браузер и редактор файлов, просмотр CLI в реальном времени, уведомления, skills, конфигурацию MCP и удалённые настольные цели.

Приложение не включает подписку на модели. Установите и аутентифицируйте каждую CLI агента, которую собираетесь использовать, на том же хосте и под тем же пользователем операционной системы, что запускает Gajae App.

### Поддерживаемые агенты

- **Gajae Code (GJC)**
- **Claude Code**
- **Cursor**
- **Codex**
- **OpenCode**

Модели, настройки усилий, режимы разрешений, история сеансов, skills и возможности MCP, специфичные для провайдера, отображаются только тогда, когда этот провайдер их поддерживает.

<a id="quick-start"></a>
## Быстрый старт

### Требования

- Node.js 22.x или 24.x
- npm и Git
- Хотя бы одна поддерживаемая CLI агента, уже установленная и аутентифицированная

### Запуск веб-приложения из исходного кода

```bash
git clone https://github.com/devswha/gajae-app.git
cd gajae-app
npm ci
npm run dev
```

Откройте <http://127.0.0.1:5173>. Бэкенд разработки слушает на `127.0.0.1:3001`.

### Запуск настольного приложения в разработке

Оставьте веб-стек работающим и запустите Electron во втором терминале.

Терминал 1:

```bash
npm run dev
```

Терминал 2:

```bash
npm run desktop:dev
```

<a id="first-run"></a>
## Первый запуск

1. **Создайте учётную запись владельца.** Откройте Gajae App и создайте единственную локальную учётную запись приложения. Имя пользователя должно содержать не менее 3 символов, пароль — не менее 6.
2. **Настройте идентичность Git.** Введите имя и адрес электронной почты для коммитов, создаваемых на этом хосте. Это записывает глобальные значения Git `user.name` и `user.email`; вход в GitHub не требуется.
3. **Подключите агентов программирования.** Завершите доступные процедуры входа провайдеров во время онбординга либо пропустите их и позже используйте **Settings → Agents**. Аутентификация CLI на уровне хоста остаётся источником истины.
4. **Добавьте проект.** Используйте действие проекта на боковой панели, чтобы выбрать существующий каталог или создать/клонировать рабочее пространство. Пути относятся к машине, на которой работает сервер, а не обязательно к устройству, отображающему браузер.
5. **Начните сеанс.** Выберите проект, доступного провайдера, настройте поддерживаемые провайдером элементы управления моделью и разрешениями, затем отправьте первый prompt.

<a id="daily-workflow"></a>
## Ежедневный рабочий процесс

### Проекты и сеансы

- Добавьте локальное рабочее пространство по абсолютному пути или клонируйте Git-репозиторий через мастер проектов.
- Сохраняйте токен GitHub в **Settings → API & Credentials** только когда он нужен для HTTPS-клонирования; SSH URL используют SSH-конфигурацию пользователя сервера.
- Разверните проект на боковой панели, чтобы возобновить проиндексированные сеансы. Gajae App читает хранилища сеансов поддерживаемых провайдеров и сохраняет идентичности провайдеров раздельными.
- Начните новый чат из выбранного проекта. Остановка запуска останавливает активный процесс агента; она не удаляет проект или его историю.

### Чат и подтверждения

- Отправляйте текст, вложения изображений, упоминания файлов и поддерживаемые провайдером slash-команды.
- Проверяйте вызовы инструментов и отвечайте на запросы разрешений в чате вместо бездумного включения неограниченного выполнения.
- Используйте элементы управления моделью, усилием, мышлением и разрешениями только там, где выбранный провайдер их предоставляет.
- Возобновляйте прежние сеансы с боковой панели. Названия сеансов можно редактировать без изменения собственных идентификаторов сеансов провайдера.

### Файлы

Откройте панель Files, чтобы просматривать настроенный корень рабочего пространства, предварительно просматривать изображения и Markdown, редактировать текстовые файлы, создавать папки и загружать файлы. Доступ к файлам ограничен проверенными путями проекта; выходы через символьные ссылки и обход каталогов отклоняются.

### Сеансы Live CLI

Gajae App может отображать поддерживаемые сеансы агентов, которые уже работают под `tmux`. Строки Live используют имя сеанса tmux, открываются как представления с терминальным доступом и остаются во владении tmux, а не веб-сервера. Перезапуск сервера не должен завершать эти внешние сеансы.

### Уведомления

Включите уведомления браузера или настольные уведомления в **Settings → Notifications**. Для событий завершения запуска, ошибок, требующихся разрешений и поддерживаемых live-turn предусмотрены отдельные элементы управления, поэтому шумные каналы можно отключать независимо.

## Удалённое использование

По умолчанию сервер привязывается к loopback. Для другого устройства сохраните эту привязку и используйте доверенный VPN или SSH-туннель:

```bash
ssh -N -L 3001:127.0.0.1:3001 user@server
```

Затем локально откройте <http://127.0.0.1:3001>. Не открывайте порт 3001 напрямую для публичного интернета.

Приложение Electron может регистрировать удалённые серверы Gajae App. Удалённые цели требуют HTTPS; обычный HTTP принимается только для точных loopback origins. Каждая локальная или удалённая цель использует изолированный раздел сеанса Electron, поэтому cookie и хранилище не являются общими.

## Установка для production

Production поддерживается на Linux x86_64 с glibc 2.35 или новее, Node.js 22 и службой systemd уровня пользователя.

Используйте неизменяемый артефакт `gajae-app-server-<version>-linux-x64-node22.tar.gz` из [GitHub Releases](https://github.com/devswha/gajae-app/releases). Поддерживаемая установка должна:

1. скачать закреплённую версию и соответствующий файл `.sha256`;
2. проверить контрольную сумму до распаковки;
3. распаковать его в `~/.gajae-app/releases/<version>`;
4. указать `~/.gajae-app/current` на этот релиз;
5. запустить `gajae-app.service` как пользовательскую службу и проверить `http://127.0.0.1:3001/health`.

Следуйте [docs/INSTALL.md](docs/INSTALL.md) для точных команд первой установки и [docs/SELF-HOST.md](docs/SELF-HOST.md) для операций службы, обновлений, удалённого доступа, отката и удаления. Не развёртывайте изменяемый URL `latest`, копию из реестра пакетов, образ контейнера или непроверенную сборку из исходного кода в качестве production-сервера.

## Устранение неполадок

| Симптом | Проверка |
|---|---|
| Провайдер недоступен | Убедитесь, что его CLI установлена, аутентифицирована и видна в `PATH` пользователя, запускающего Gajae App; затем повторно проверьте **Settings → Agents**. |
| Путь проекта отклонён | Введите абсолютный путь, который существует на хосте сервера и доступен пользователю сервера. |
| Electron в разработке открывает пустую или неудачную страницу | Оставьте `npm run dev` работающим до запуска `npm run desktop:dev`. |
| Служба не запускается | Выполните `systemctl --user status gajae-app.service` и `journalctl --user -u gajae-app.service -f`. |
| Удалённый доступ не работает | Сначала подтвердите локальную конечную точку `/health`, затем проверьте маршрут SSH/VPN или зарегистрированный HTTPS origin. |
| Старые учётные данные всё ещё недействительны после входа | Повторно подключите провайдера в **Settings → Agents** и проверьте CLI непосредственно под пользователем службы. |

## Команды разработки

| Команда | Назначение |
|---|---|
| `npm run dev` | Запустить клиент Vite и бэкенд разработки |
| `npm run server:dev` | Запустить только бэкенд разработки |
| `npm run client` | Запустить только клиент Vite |
| `npm run desktop:dev` | Запустить Electron с клиентом разработки |
| `npm test` | Запустить тесты сервера, клиента и Electron |
| `npm run typecheck` | Проверить типы клиента и сервера |
| `npm run lint` | Запустить ESLint для кода продукта и инструментов |
| `npm run check:identity` | Проверить правила идентичности продукта, юридической информации и происхождения |
| `npm run build` | Собрать production-клиент и сервер |
| `npm run verify` | Запустить полный барьер релиза |

Используйте Node.js 22 или 24 и запускайте полный барьер перед отправкой изменений:

```bash
npm run verify
```

Он запускает аудит зависимостей, проверки типов, все разделы тестов, lint, проверку идентичности и production-сборки.

## Границы безопасности и данных

- Веб-аутентификация использует cookie `HttpOnly`, `SameSite=Strict` с постоянной отменой при выходе.
- Учётные данные не принимаются из параметров запроса URL. Внешние API-ключи агентов используют заголовок `X-API-Key`.
- Файлы проекта разрешаются через проверки канонического пути и символьных ссылок; записи используют атомарную замену в том же каталоге.
- Загрузки используют частные временные каталоги для каждого запроса, которые очищаются после завершения или ошибки.
- Electron по умолчанию запрещает разрешения целей и ограничивает IPC зарегистрированными фреймами launcher.
- Создавайте резервную копию `~/.gajae-app/data` перед обновлениями или миграцией хоста. Переключения релизов должны сохранять этот каталог.
